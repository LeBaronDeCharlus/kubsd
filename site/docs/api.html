HTTP API reference

Both APIs speak YAML request/response bodies over HTTP/1.1, parsed with a small hand-rolled parser (httparse), no async runtime.

keel-agentd (Unix socket)

Default socket: /var/run/keel-agentd.sock.

Method & pathBodySuccessErrors
PUT /jails/<name>JailSpec YAML; metadata.name must match <name>200, reconciled immediately before responding400 invalid YAML or name/body mismatch, 409 attempted change to an immutable field
GET /jailsnone200, YAML list of every tracked jail—
GET /jails/<name>none200, YAML for that jail404 unknown jail
DELETE /jails/<name>none200, jail/dataset/network/limits torn down before responding404 unknown jail

keel-controlplane (TCP)

Nodes live on separate hosts, so this API is served over a TcpListener instead of a Unix socket.

Method & pathBodySuccessErrors
POST /nodes/registerYAML: id, addr200, node registered or re-registered400 invalid YAML
POST /nodes/<id>/heartbeatnone200, node's last-seen time updated404 unknown node (the node should re-register)
GET /nodesnone200, YAML list of every node: id, addr, computed status (Alive within 15 seconds of the last heartbeat, otherwise Dead), and last_seen_secs—

Liveness is computed on every read rather than tracked by a background sweep. There is no persistence: a control-plane restart forgets every node, and each node notices on its next heartbeat and re-registers automatically.
