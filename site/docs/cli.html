keelctl CLI reference

keelctl talks to keel-agentd's local HTTP API over its Unix socket.

Global flag

FlagDefaultMeaning
--socket PATH/var/run/keel-agentd.sockPath to the keel-agentd Unix socket to talk to.

keelctl apply -f FILE

Reads a JailSpec YAML file, validates it locally, and sends it as a PUT /jails/<name> request (the name comes from the spec's own metadata.name).

keelctl apply -f jail.yaml

keelctl get [name]

With no argument, lists every jail keel-agentd is tracking (GET /jails). With a name, fetches just that one (GET /jails/<name>).

keelctl get
keelctl get web-1

keelctl delete NAME

Tears down the named jail, its ZFS dataset, its network attachment, and its resource limits (DELETE /jails/<name>).

keelctl delete web-1

Errors

On any non-2xx response, keelctl prints the server's error message to stderr and exits non-zero. See the HTTP API reference for the specific error cases each endpoint can return.
